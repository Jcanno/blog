    # HTTP/3

    由于 TCP 有丢包重发的机制，比如客户端通过 TCP 发送两个包给服务端，第一个包丢了，服务端就暂时缓存了第二个包，等待第一个包的重发，所以在 TCP 层就有了队头阻塞的问题，HTTP 层不能彻底解决这个问题

    因此 HTTP/3 不再依赖 TCP，QUIC 就选定了 UDP，在它之上把 TCP 的那一套连接管理、拥塞窗口、流量控制等“搬”了过来，“去其糟粕，取其精华”，打造出了一个全新的可靠传输协议，可以认为是“新时代的 TCP”。
